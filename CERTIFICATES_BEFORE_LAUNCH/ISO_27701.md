# 🏢 ISO/IEC 27701

> Privacy extension to ISO 27001 (a PIMS). Relevant for demonstrating GDPR-grade privacy management to enterprise/EU customers — deferred.

| Field | Detail |
|-------|--------|
| **Overview** | Privacy Information Management System standard extending ISO 27001 for PII processing. |
| **Purpose** | Demonstrate systematic privacy management (maps to GDPR). |
| **Category** | 🏢 Enterprise Only |
| **Why it is needed** | Enterprise/EU buyers value it; **not** needed for launch. |
| **Legally required?** | No. |
| **Technically required?** | No. |
| **When to implement** | After ISO 27001, for privacy-sensitive enterprise/EU deals. |
| **Priority** | 🏢 Deferred (enterprise) |
| **Estimated Cost** | Incremental on top of ISO 27001. |
| **Renewal** | With the ISO 27001 cycle. |
| **Official Website** | https://www.iso.org/standard/71670.html |
| **Eligibility** | An ISO 27001 ISMS in place. |

## Step-by-Step Process
1. Establish ISO 27001 first.
2. Extend the ISMS with PIMS controls (data subject rights, PII handling).
3. Audit + certify as a 27001 extension.

## Required Documents
- ISO 27001 docs + privacy controls + records of processing.

## Implementation Guide
- Reuse [GDPR](GDPR.md) + [../docs/COMPLIANCE.md](../docs/COMPLIANCE.md) work as PIMS evidence.
