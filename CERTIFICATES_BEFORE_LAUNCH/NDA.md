# 🤐 NDA (Non-Disclosure Agreement)

> Protects CodeVault's confidential information when working with contractors, partners, vendors, or enterprise prospects.

| Field | Detail |
|-------|--------|
| **Overview** | A contract binding parties to keep shared confidential information secret. |
| **Purpose** | Safeguard proprietary code, roadmap, security details, and customer data discussions. |
| **Category** | 🔵 Required During Scaling |
| **Why it is needed** | As you hire/partner, you'll share sensitive material (architecture, secrets handling, deals). |
| **Legally required?** | No. |
| **Technically required?** | No. |
| **When to implement** | When engaging contractors/partners/enterprise pilots. |
| **Priority** | 🔵 Medium (when sharing externally) |
| **Estimated Cost** | $0 (template) → minor (counsel review). |
| **Renewal** | Per engagement; defined term. |
| **Official Website** | (template sources) commonpaper.com, ironcladapp.com |
| **Eligibility** | N/A. |

## Step-by-Step Process
1. Choose mutual vs one-way NDA per relationship.
2. Define confidential info, term, permitted use, exclusions.
3. Execute before sharing sensitive material.

## Required Documents
- NDA template (mutual + one-way variants).

## Implementation Guide
- Use a standard mutual NDA for partners; one-way for contractors receiving CodeVault info.

## Best Practices
- Sign before disclosure; reasonable term; clear definition of "confidential".

## Common Mistakes
- Sharing first, signing later; overbroad/unenforceable terms.

## CodeVault-specific Notes
- Especially relevant before sharing **security architecture** or token-handling details, or during enterprise pilots/audits.

## Future Considerations
- Standard contractor agreement incorporating NDA + IP assignment.

## Checklist
- [ ] Mutual + one-way NDA templates ready
- [ ] Signed before sensitive disclosure
- [ ] Term + scope defined
- [ ] Records retained
