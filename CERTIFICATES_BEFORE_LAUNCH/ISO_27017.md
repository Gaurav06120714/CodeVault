# 🏢 ISO/IEC 27017

> Cloud-security controls extension to ISO 27001. Relevant once CodeVault's cloud posture must be formally attested to enterprise buyers — deferred.

| Field | Detail |
|-------|--------|
| **Overview** | Code of practice for information security controls specific to cloud services. |
| **Purpose** | Address cloud-specific risks (shared responsibility, virtualization, admin ops). |
| **Category** | 🏢 Enterprise Only |
| **Why it is needed** | Cloud-focused enterprise buyers may request it; **not** needed for launch. |
| **Legally required?** | No. |
| **Technically required?** | No. |
| **When to implement** | With ISO 27001, for cloud-conscious enterprise deals. |
| **Priority** | 🏢 Deferred (enterprise) |
| **Estimated Cost** | Incremental on ISO 27001. |
| **Renewal** | With the 27001 cycle. |
| **Official Website** | https://www.iso.org/standard/43757.html |
| **Eligibility** | ISO 27001 ISMS. |

## Step-by-Step Process
1. Establish ISO 27001.
2. Add 27017 cloud controls (shared responsibility, segregation, monitoring).
3. Certify as an extension.

## Required Documents
- ISO 27001 docs + cloud control evidence.

## Implementation Guide
- Map to [../docs/CLOUD_SECURITY.md](../docs/CLOUD_SECURITY.md) + [../docs/INFRASTRUCTURE_SECURITY.md](../docs/INFRASTRUCTURE_SECURITY.md) (Cloudflare/Vercel/managed DB).

- Clarify the shared-responsibility split with each cloud provider; document admin operations.
